# Commands

The console runs the same `@Command` handlers over **stdin** (the server's terminal) and
over the **HTTP remote console** (`:5090`, configurable in `config.json`). A leading `/`
is optional. Type `help` for the live list. Commands target the **active player** — the
most recently registered profile (i.e. the live client once it connects), falling back to
the auto-created default player offline.

| Command | Aliases | Usage | Description |
|---------|---------|-------|-------------|
| `give` | `g` | `give gem <amount>` · `give <itemMasterId> [xN]` | Grant gems or an item to the active player (live state mutation). |
| `giveall` | `ga` | `giveall` | Max gems on the active player + report full inventory. |
| `chart` | `music`, `song` | `chart <lock\|unlock\|clear\|done\|fc\|ap\|afc\|reset> <musicId \| all> [diff:<easy\|normal\|hard\|expert\|all>] [hscore:<int>]` | Set a song's lock + clear state (+ optional difficulty / high score) on the active player. |
| `help` | `?` | `help` | List available commands. |

### `/chart` states

`/chart <state> <musicId \| all> [diff:<easy\|normal\|hard\|expert\|all>] [hscore:<int>]` — `all` targets every `MasterLiveMusic` song; a clear state also unlocks the song. Clear ranks map to the in-game `ClearStatus`. By default a state applies to **every difficulty** (Easy=0/Normal=1/Hard=2/Expert=3); `diff:<…>` limits it to one. The optional `hscore:<int>` sets the high score on every affected chart (clear states only; default is derived from the note count):

| State | Effect |
|-------|--------|
| `lock` | Song **locked** — removed from owned songs; its scores cleared. |
| `unlock` | Song **unlocked**, uncleared — owned, no clear record. |
| `reset` | **Never-played** — scores wiped (high score 0); ownership unchanged; `hscore` ignored. |
| `done` | Every chart marked **Failed (1)** — played/attempted. |
| `clear` | Every chart marked **Clear (2)**. |
| `afc` | Every chart marked **Assist Full Combo (3)** |
| `fc` | Every chart marked **Full Combo (4)**. |
| `ap` | Every chart marked **All Perfect (5)**. |

Examples: `/chart ap all`, `/chart fc 100001 hscore:1850000`, `/chart ap 100001 diff:expert`, `/chart lock 100007`, `/chart reset all diff:hard`.

> This table is generated by hand from the `@Command` annotations; the **authoritative,
> always-current** list (with every item / song / card id) is the GM Handbook — see below.
> Item and music ids come from `MasterItem` / `MasterLiveMusic`.

## GM Handbook

```powershell
.\gradlew.bat generateHandbook                 # → "GM Handbook/GM Handbook - EN.txt"
.\gradlew.bat generateHandbook --args="CHS"    # also: JP, CHT, KR
```

The handbook lists every console command plus `id → name` maps for items, songs (live
music), member cards, support cards, characters, bands, band items, stamps, and the
lane / note / effect / SE skins. Names resolve through the `MasterText` table (Japanese /
English / Traditional-Chinese / Simplified-Chinese / Korean). CBT1 has almost all texts
in japanese only, hence names fall back to the original Japanese title when no translation
is available — so e.g. `generateHandbook --args="EN"` still shows Japanese song titles where
no English exists.
